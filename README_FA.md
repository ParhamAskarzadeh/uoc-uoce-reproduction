# گزارش و بازتولید مقاله (UOC - LDK 2025)

**هدف این مخزن:** بازتولید حداقل یک نتیجه از ریپوی اصلی، اجرای یک بهبود معنادار، و ارائه دمو روی CPU بدون دست‌کاری ریپوی اصلی.

**ساختار پروژه**
- `project/original_repo`: کپی بدون تغییر از ریپوی اصلی
- `project/experiments`: اسکریپت‌های بازتولید و بهبود
- `project/demo`: نوت‌بوک دمو اجرای سبک روی CPU
- `project/README_FA.md`: راهنمای فارسی
- `project/experiments_FA.md`: توضیح آزمایش‌ها و پیاده‌سازی
- `project/paper_summary_FA.pdf`: خلاصه تحلیلی ۲–۳ صفحه‌ای

**بازتولید نتیجه (اجباری)**
- اجرای بازتولید (Tuple-Level Exact Match):
  - `python3 project/experiments/reproduce_tuple_em.py`
- خروجی به‌صورت JSON در `project/experiments/baseline_results.json` ذخیره می‌شود.
- خلاصه نتایج بازتولید شده:
  - MVP / ASTE: `F1 = 0.3475`
  - GEN_SCL_NAT / ASTE: `F1 = 0.3484`
  - MVP / ACOS: `F1 = 0.1348`
  - GEN_SCL_NAT / ACOS: `F1 = 0.0345`

**بهبود ما (معنادار و قابل مشاهده)**
- ایده: در محاسبه Tuple-Level، رشته‌ی «موجودیت–دسته» اگر چندواژه‌ای باشد، در کد اصلی به چند بخش شکسته می‌شود و آرایش فیلدها به هم می‌ریزد. ما یک پارسِرِ ساده اعمال کردیم که فقط روی **اولین فاصله** جدا می‌کند تا دسته‌ی چندواژه‌ای حفظ شود.
- اجرای آزمایش بهبود:
  - `python3 project/experiments/entity_category_fix_eval.py`
- نتیجه:
  - MVP / ASTE: `F1` از `0.3475` به `0.3546` افزایش یافت.

**دموی سریع (روی CPU)**
- نوت‌بوک: `project/demo/uoc_demo.ipynb`
- در این دمو، خروجی‌های از پیش محاسبه‌شده‌ی یکی از مدل‌ها بارگذاری می‌شود و برای یک نمونه، «پیش‌بینی» نمایش داده می‌شود.

**نیازمندی‌ها**
- Python 3.11+ (در این پروژه با Python 3.12 اجرا شده است)
- بدون وابستگی خارجی؛ همه‌ی اسکریپت‌ها با کتابخانه استاندارد اجرا می‌شوند.

**ساختار ارائه ویدیویی ۱۰–۱۵ دقیقه‌ای**
1. مسئله و اهمیت استخراج مؤلفه‌های نظر
2. مرور روش و ایده UOC
3. نمای کلی ریپو و داده‌ها
4. فرآیند بازتولید و نتایج اصلی
5. بهبود پیشنهادی ما و انگیزه آن
6. نتایج جدید و مقایسه
7. دمو اجرای سبک روی CPU
8. نقد علمی و محدودیت‌ها
9. کاربرد در سیستم واقعی

**یادداشت مهم**
- ریپوی اصلی بدون تغییر نگه داشته شده است و تمام تغییرات در مسیر `project/` قرار دارد.
